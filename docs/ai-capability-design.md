# Taonix AI 能力設計 (AI Capability Design)

> **版本：v9.0.0+**
> **目標：實現具備推理深度與環境感知的智慧助理**

## 一、 推理深度 (Blackboard Reasoning)
Taonix 的 AI 能力不再僅限於單次 Prompt 響應，而是跨 Agent 的推理協作。
- **思考痕跡注入**: 每次呼叫 Agent 時，自動從黑板提取先前的「推理節點」。
- **假說驗證**: 支持 Agent 在黑板發布假設 (Hypotheses)，並由其他專家 Agent 進行驗證與證偽。

## 二、 元演進能力 (Meta-Evolutionary AI)
系統不再受限於預設的功能集。
- **自發學習**: 偵測到現有技能無法滿足需求時，AI 建築師自動設計新算法。
- **循環修復**: AI 具備自我 Debug 生成代碼的能力，直到通過沙盒 Dry Run 驗證。

## 三、 環境感知執行 (Environment-Aware AI)
AI Agent 具備對「物理世界」的感知力。
- **狀態快照比對**: 自動比對配置、進程與檔案系統狀態。
- **自癒決策**: 偵測到環境異常（如僵屍進程）時，AI 能主動制定清理方案。

## 四、 長程規劃與編排
- **任務狀態機**: 使用持久化 JSON 跟蹤複雜工程的每一步。
- **上下文恢復**: AI 能從斷點續傳，找回數小時前的推理上下文。

---
## 技術選型 (v9.0)
- **核心模型**: 優先採用 Gemini-3-Flash 或更高階模型進行邏輯鏈路推理。
- **本地回退**: 支持 Ollama 調用本地模型處理數據敏感型任務。
- **通訊介面**: Model Context Protocol (MCP) v1.9.0 標準。
